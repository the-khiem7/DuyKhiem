## Overview

EzyFix is a home-repair marketplace backend centered on operational workflow. The product needs more than listing and booking. It has to manage service requests, technician coordination, payment states, disputes, and user trust across the full request lifecycle.

## Core Responsibilities

- Modeled request lifecycle transitions across multiple user roles.
- Supported payout and escrow-like flows tied to service progress.
- Designed backend behavior with dispute and exception handling in mind.
- Worked with real-time updates where status changes matter immediately.

## Why This Project Is Important

EzyFix highlights the type of backend work I care about: multi-step workflows where correctness, transparency, and maintainability matter just as much as feature speed.
